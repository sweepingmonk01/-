# 基础科学双引擎新增方案

更新日期：2026-04-24

## 一句话结论

新增“物理学”和“神经科学”是必要的，但不应按普通年级课程处理，而应作为列子御风的基础科学探索层。

它们的产品定位是：

- `物理学 / World Engine`：帮助学生理解外部世界的基本规律，训练因果、模型、尺度、守恒、对称性、系统思维。
- `神经科学 / Mind Engine`：帮助学生理解自己的学习系统，训练注意、记忆、情绪、决策、错因修复和 AI 协作能力。

这两门课不是“新增两个频道”，而是把现有错题本、知识图谱、AI Active 状态引擎和生成式漫剧系统升级为跨年级、跨课本的元知识底座。

## 1. 产品定位

### 1.1 当前产品边界

列子御风当前主链路是：

```text
错题 / 作业输入
-> 错因诊断
-> 知识点与知识动作
-> AI Active 策略调度
-> GPT-image-2 卡图 / Seedance 知识点漫剧
-> 交互裁决
-> 状态向量更新
```

这个闭环适合解决“当前题目为什么错、下一步怎么修复”。

新增基础科学双引擎后，主链路升级为：

```text
错题 / 作业输入
-> 课本知识点
-> 基础科学原理
-> 跨学科模型
-> 个体认知状态
-> AI Active 干预策略
-> 可视化拓扑卡 / 漫画分镜 / 探索任务
-> 状态与能力迁移
```

核心变化是：系统不再只围绕年级和教材组织学习，而是先建立全局框架，再回落到局部题目。

### 1.2 新范式定义

推荐产品表述：

> 基础科学双引擎是列子御风的元知识探索层：World Engine 解释世界如何运行，Mind Engine 解释学习者如何理解世界。

这能把产品从“AI 错题与提分工具”提升为“AI 时代的基础科学探索系统”。

## 2. 信息内核图文拓扑卡

用户提供的两张 GPT-image-2 图适合作为后续标准样板：

- `/Users/yucheng/Desktop/物理学基础框架.png`
- `/Users/yucheng/Desktop/神经科学基础框架.png`

这类卡片不是普通海报，而是信息内核图文拓扑卡，应该承担四个功能：

1. **全局导航**：学生先看到整门学科的结构，再进入局部知识点。
2. **知识压缩**：每张卡压缩一门学科的核心对象、机制、路径和目标。
3. **模型锚点**：作为知识图谱根节点和 GPT-image-2 / Seedance 生成素材的视觉参照。
4. **跨学科连接**：用边关系连接数学、物理、生物、AI、心理学、考试题目和现实问题。

## 3. 课程层级设计

建议新增一级课程域：

```text
基础科学探索
├── World Engine：物理学基础框架
└── Mind Engine：神经科学基础框架
```

每个课程域采用五层结构：

1. **全局地图层**
   展示整门学科的拓扑卡和 8-10 个一级节点。

2. **核心对象层**
   明确这门科学研究什么对象。
   物理学：物质、能量、时空、相互作用、信息。
   神经科学：神经元、突触、回路、脑区、行为、认知。

3. **基本机制层**
   解释规律如何发生。
   物理学：守恒、对称性、场、统计、量子化、时空弯曲。
   神经科学：电生理、递质、可塑性、反馈回路、预测加工。

4. **探索任务层**
   让学生完成拖拽、排序、因果链判断、模型选择、知识动作执行。

5. **课本与错题连接层**
   把基础科学原理映射回学生当前教材、题目和错因。

## 5. 图谱数据模型建议

建议不要把基础科学节点混进普通教材目录，而是给知识节点增加 `domainLayer` 和 `scale`。

```ts
export type KnowledgeDomainLayer =
  | 'curriculum'
  | 'foundation-science'
  | 'meta-cognition'
  | 'application';

export type FoundationScienceDomain = 'physics' | 'neuroscience';

export interface FoundationKnowledgeNode {
  key: string;
  label: string;
  domain: FoundationScienceDomain;
  domainLayer: 'foundation-science';
  scale: 'global' | 'module' | 'concept' | 'mechanism' | 'task';
  summary: string;
  coreQuestion: string;
  prerequisites: string[];
  relatedCurriculumNodes: string[];
  relatedMistakePatterns: string[];
  mediaPromptSeed: {
    visualMetaphor: string;
    gptImageStyle: string;
    seedanceStoryboard: string[];
  };
}
```

建议边关系先固定为：

```ts
export type KnowledgeEdgeKind =
  | 'prerequisite'
  | 'explains'
  | 'analogy'
  | 'contrasts'
  | 'applies-to'
  | 'supports-cognition'
  | 'maps-to-curriculum'
  | 'maps-to-mistake';
```

关键点：

- `physics.symmetry_conservation -> maps-to-curriculum -> math.equation_balance`
- `neuroscience.learning_memory -> supports-cognition -> ai_active.signal_noise_ratio`
- `physics.thermo_stat -> analogy -> neuroscience.population_coding`
- `neuroscience.cognition_consciousness -> maps-to-mistake -> careless_reading`

## 6. 生成媒体规范

### 6.1 GPT-image-2 拓扑卡规范

基础科学拓扑卡统一使用：

```text
Generate an information-dense educational topology card for a mobile learning product.
Subject: {物理学 / 神经科学}.
Show a central title node, 8-10 numbered module cards, cross-module arrows, concept icons, formula or signal fragments, and a bottom legend.
The card should help a student see the whole discipline before entering local textbook chapters.
Avoid real people, copyrighted characters, school logos, private student information, unreadable dense paragraphs.
Style: luminous scientific atlas, crisp Chinese labels, high contrast, structured visual hierarchy, concept graph topology.
```

### 6.2 Seedance 知识点漫剧规范

基础科学视频不应生成泛科普短片，而应生成“知识点漫剧 + 探索任务”：

```text
Create a 9:16 educational manga-drama video for a foundation science exploration module.
Foundation domain: {physics | neuroscience}.
Core node: {nodeLabel}.
Core question: {coreQuestion}.
Connect this foundation node to the student's current curriculum node: {curriculumNode}.
Use the provided GPT-image-2 topology card or concept card as the first-frame visual reference.
Storyboard:
1. Global map: show where this node sits in the whole discipline.
2. Mechanism scene: animate the core mechanism with symbolic props.
3. Curriculum bridge: connect the mechanism to a familiar textbook problem.
4. Exploration action: ask the student to choose, drag, sequence, or explain one key relation.
Avoid generic scenery. Every shot must teach, test, or connect the foundation concept.
```

## 7. 前端入口建议

建议新增一个“探索”入口，而不是塞进错题本。

信息架构：

```text
底部导航 / 首页任务区
└── 探索
    ├── 基础科学双引擎
    │   ├── World Engine 物理学
    │   └── Mind Engine 神经科学
    ├── 今日跨学科连接
    ├── 我的全局地图
    └── 从错题进入基础原理
```

首屏不做课程列表，直接展示两张拓扑卡：

- 左：World Engine，标题“世界如何运行”
- 右：Mind Engine，标题“我如何学习”

点击后进入对应的全局地图，再进入节点漫剧和探索任务。

## 8. 与 AI Active 内核的连接

物理学主要服务“外部世界模型”：

- 强化因果链
- 强化尺度转换
- 强化模型选择
- 强化守恒与约束思维
- 强化从现象到公式的抽象能力

神经科学主要服务“学习者自我模型”：

- 解释注意为什么会丢失
- 解释重复为什么能巩固记忆
- 解释错因为什么会形成习惯回路
- 解释情绪和疲劳为什么影响推理
- 解释 AI Active 的时间、信噪比、情绪三轴

建议将 Mind Engine 直接映射到认知雷达：

```text
time -> 执行节奏 / 工作记忆负荷 / 疲劳调节
signalNoiseRatio -> 注意选择 / 错因噪声 / 规则信号清晰度
emotion -> 奖励系统 / 挫败恢复 / 持续行动意愿
```

这样“认知雷达”不再只是 UI 指标，而有了神经科学解释层。

## 9. 工程落地顺序

### P1：文档与种子数据

1. 固化本方案。
2. 新增 `foundation-science` domain 类型。
3. 新增物理学、神经科学首批 18 个一级节点。
4. 给每个节点写 `summary`、`coreQuestion`、`mediaPromptSeed`。

### P2：前端探索入口

1. 新增基础科学双引擎入口。
2. 展示两张 GPT-image-2 拓扑卡。
3. 支持点击一级节点查看说明、关联课内知识和错因。

### P3：图谱接入错题链路

1. 在错题诊断后推荐一个基础科学解释节点。
2. 在知识图谱中显示“课本知识点 -> 基础科学原理”的边。
3. 允许学生从错题跳转到基础科学漫剧。

### P4：生成式媒体闭环

1. GPT-image-2 生成节点概念卡。
2. Seedance 生成 6-10 秒知识点漫剧。
3. 学生完成探索动作后，写入 learning cycle。
4. 状态向量根据探索结果更新。

## 10. 验收标准

第一阶段完成标准：

- 产品中能看到“基础科学探索”域。
- 物理学和神经科学各有不少于 9 个一级节点。
- 每个节点能被知识图谱查询。
- 每个节点都有 GPT-image-2 卡图 prompt seed 和 Seedance storyboard seed。
- 至少一条错题链路能映射到基础科学节点。

第二阶段完成标准：

- 学生能从一张拓扑卡进入具体节点。
- 节点页能展示“全局位置、核心问题、课内连接、错因连接、探索任务”。
- 至少 2 个节点能生成真实卡图或视频。
- learning cycle 能记录基础科学探索行为。

## 11. 当前任务判断

短期不能让基础科学双引擎打断当前 P0 闭环工程。

更合理的顺序是：

1. 继续完成学习闭环、状态向量、策略调度和媒体任务稳定性。
2. 同步把基础科学双引擎作为 `P1/P2` 种子体系接入。
3. 先让它成为可查询、可展示、可生成媒体的知识图谱层。
4. 再逐步让它参与错题诊断和 AI Active 策略调度。

最终目标不是多做两门课，而是让列子御风拥有一个能突破年级和课本边界的元知识底座。
